# GS215 Modbus Simulator 使用说明

这是一个 WPF 桌面程序，用于模拟 Modbus 从站设备、主站轮询/写入、协议导入、寄存器检视和 API 数据比对。

## 1. 登录账号

程序启动后会先进入登录界面。

| 角色 | 用户名 | 密码 | 权限 |
| --- | --- | --- | --- |
| 管理员 | `admin` | `dy@2026..` | 所有功能均可操作 |
| 普通用户 | `user` | `user123` | 不能删除/编辑部分关键配置 |

普通用户限制：

- 从站不能删除协议。
- 从站不能编辑协议。
- 主站不能删除所选站点。

## 2. 从站功能使用

从站用于模拟设备协议，响应工控机或主站的 Modbus RTU/TCP 读写请求。

### 2.1 监听配置

顶部区域是监听配置。

常用字段：

- 协议：`Tcp` 或 `Rtu`。
- TCP 监听地址：如 `0.0.0.0` 或本机网卡 IP。
- TCP 端口：常用 `502`。
- RTU 串口：如 `COM3`。
- RTU 波特率：如 `9600`。
- Slave ID：从站地址。
- 勾选框：只有勾选后，点击启动时才会参与监听。

常用操作：

- `刷新DB`：重新加载数据库中已保存的协议设备。
- `↻`：刷新本机 IP 和串口列表。
- `＋ 新增监听`：新增一条监听配置。
- `▶ 启动全部`：启动已勾选的监听配置。
- `⏹ 停止全部`：停止所有正在运行的监听。
- 单条监听右侧按钮：启动/停止当前监听。

注意：

- 设备列表和协议导入设备区域，只有勾选的设备协议才会参与监听。
- 寄存器检视区域不需要勾选设备，进入寄存器检视后只使用寄存器检视自己的数据进行监听。
- 已启动监听时，切换设备不会自动切换监听配置，避免运行中的监听被误改。

### 2.2 设备列表区域

左侧 `设备列表(ing)` 是内置设备协议，例如 PCS、BMS、MPPT、STS、电表、除湿机、温控设备等。

使用流程：

1. 在设备列表中选择设备协议。
2. 勾选需要启动的设备。
3. 配置顶部监听参数。
4. 点击 `▶ 启动全部` 或单条监听启动。
5. 工控机按对应 Slave ID、串口/IP、端口访问即可。

### 2.3 协议导入设备区域

`协议导入设备` 用于显示用户从 Excel 或寄存器检视保存出来的协议设备。

常用操作：

- 双击协议名称：修改设备名称并保存到数据库。
- `新建协议`：导入新的协议配置。
- `导出协议`：导出当前选中的协议设备，字段格式为地址、中文名、英文名、单位、范围、描述。
- `删除协议`：删除当前选中的协议设备，普通用户不可操作。
- `编辑协议`：把当前协议加载到寄存器检视中编辑，保存后替换原协议，普通用户不可操作。
- `＋ 新增行`：新增寄存器行。
- `搜索`：按中文名或英文名过滤。
- `随机生成`：按最小值/最大值给勾选行生成随机当前值。
- `全选`：勾选/取消勾选表格行。
- `立即比对`：根据 API 和 Token 比对当前值。
- `定位未通过`：跳转到未通过 API 比对的寄存器行。

表格字段：

- 地址
- 中文名
- 英文名
- 当前值
- 单位
- 范围
- 描述

当前值支持：

- 十进制
- 二进制
- 十六进制

在当前值单元格右键可以切换显示/输入模式。当前值修改后会写入寄存器，并用于从站响应主机读取。

### 2.4 API 比对

协议导入设备区域支持 API 数据比对。

使用步骤：

1. 在设备名称旁填写 API 地址。
2. 填写 Token / Authorization。
3. 填写允许误差。
4. 点击 `立即比对`。
5. 通过的行显示绿点，未通过显示灰点。

说明：

- 用户也可以手动点击圆点切换通过/未通过。
- API 比对不会清除之前已经通过的绿点。
- 日志会记录比对结果。

### 2.5 寄存器检视

寄存器检视用于临时添加寄存器地址、填写协议字段、实时写值、保存为协议设备。

常用操作：

- `批量加载`：按起始地址和数量加载寄存器地址，最大支持 `65535`。
- `添加单地址`：添加一个指定寄存器地址。
- `删除选中`：删除当前选中的记录行。
- `刷新全部`：从寄存器银行重新读取当前值。
- `保存为协议设备`：保存到数据库，并显示到协议导入设备区域。
- `清空`：清空寄存器检视，需要输入密码确认。

保存为协议设备时：

- 必填：设备名称。
- 寄存器行必填：地址、中文名。
- 选填：英文名、当前值、单位、范围、描述。
- 协议类型、Slave ID、监听地址、端口使用上方默认监听配置。

编辑体验：

- 单击字段可以编辑。
- Ctrl 可进入多选模式。
- 支持从 Excel/表格复制多行多列后粘贴到任意选中区域。
- 单个单元格内部换行不会被拆成多行记录。
- 当前值默认 `0`。
- 当前值写入后会实时写入寄存器，供工控机读取验证。

### 2.6 从站日志说明

底部运行日志用于观察监听和 Modbus 请求。

常见日志：

- `RTU FC03 addr=xxx qty=xx 来源=COM3`：主机读取保持寄存器。
- `RTU FC16 addr=xxx qty=xx 来源=COM3`：主机写多个寄存器。
- `TCP FC03 addr=xxx qty=xx 来源=ip:port`：TCP 主机读取。
- `[DEV-ERR] ... no-response ...`：设备协议长时间没有主机请求，提示检查连接、查询和链路。

说明：

- FC03 是主机读取从站数据。
- FC16 是主机向从站写入数据。
- 从站本身不是主动轮询设备，而是等待主机访问。

## 3. 主站功能使用

主站用于连接真实设备或从站模拟器，读取遥测寄存器、写入遥控寄存器，并支持站点配置保存和 API 比对。

### 3.1 连接参数

顶部连接栏字段：

- 协议：`Tcp` 或 `Rtu`。
- TCP 目标 IP。
- TCP 端口。
- RTU 串口。
- 波特率。
- Slave ID。
- 轮询间隔(ms)。

操作：

- `▶ 开始轮询`：连接设备并开始轮询。
- `⏹ 断开连接`：停止轮询并断开。
- IP 或串口旁边的 `↻`：刷新本机 IP 或串口列表。

### 3.2 站点配置

左侧是已保存站点列表。

常用操作：

- `刷新DB`：连接数据库并加载站点。
- `新建配置`：创建新的站点配置和寄存器配置。
- `删除所选站点`：删除当前选中的站点，普通用户不可操作。
- 站点名称输入框：选中站点后可修改名称，失焦自动保存。

新建配置需要填写：

- 站点名称
- 协议类型
- 目标 IP/端口或串口/波特率
- Slave ID
- 轮询间隔
- 遥测/遥控寄存器行

### 3.3 遥测表格

遥测页用于只读轮询。

主要字段：

- 中文名
- 变量名
- 地址
- 数量
- 原始寄存器
- 读取值
- 单位
- 取值范围
- 比例系数
- 更新时间

读取值右键支持切换解析方式：

- uint 无符号整数
- int 有符号整数
- ASCII 字符串

### 3.4 遥控表格

遥控页用于写入寄存器。

使用方法：

1. 先启动主站连接。
2. 在遥控表格中编辑当前值。
3. 输入框失焦后自动写入设备。
4. 日志显示写入地址和值。

写入说明：

- 遥控写入统一使用 FC16。
- 写入失败会在主站日志中显示错误原因。

### 3.5 API 比对和绿点

主站支持通过 API 对比读取值。

使用步骤：

1. 填写 API 地址。
2. 填写 Authorization。
3. 设置误差。
4. 点击 `立即比对`。
5. 通过的行显示绿点。
6. 点击 `定位未通过` 可以跳转到未通过行。

其他操作：

- `清除绿点`：清空当前站点绿点状态，需要密码。
- 可以手动点击圆点切换通过/未通过。

### 3.6 搜索和导出

搜索：

- 在搜索框输入中文名或变量名。
- 点击 `查找下一个`。
- 支持循环定位。

导出：

- 点击 `导出数据`。
- 导出当前主站轮询结果到 Excel。

日志：

- `清空`：只清空界面日志。
- `删除日志文件`：删除本地日志文件，需要密码。

## 4. 常见问题

### 4.1 TCP 监听地址填什么？

- `0.0.0.0` 表示监听本机所有网卡。
- 如果工控机访问不到，可以改成本机实际网卡 IP，例如 `172.x.x.x` 或 `192.168.x.x`。

### 4.2 RTU 没有反应怎么办？

检查：

- 串口号是否正确。
- A/B 线是否接反。
- 波特率、校验位、停止位是否一致。
- Slave ID 是否一致。
- 工控机是否真的在发 FC03/FC16 请求。

### 4.3 为什么没勾选就不监听？

从站设备列表和协议导入设备区域采用手动勾选机制。只有勾选的设备协议才会参与监听，避免误启动其他协议。

### 4.4 普通用户为什么看不到删除/编辑？

这是权限控制。普通用户不能执行危险操作，管理员登录后可以看到并使用完整功能。

### 4.5 打包后运行需要什么？

如果使用轻量发布，目标电脑需要安装 `.NET 8 Desktop Runtime`。

如果使用自包含发布，目标电脑不需要安装运行时，但包体会更大。

## 5. 开发常用命令

构建：

```powershell
dotnet build E:\SimulatorApp\SimulatorApp.csproj -c Debug --no-restore
```

轻量发布：

```powershell
dotnet publish E:\SimulatorApp\SimulatorApp.csproj -c Release -r win-x64 --self-contained false -o E:\SimulatorApp\publish-light
```

自包含发布：

```powershell
dotnet publish E:\SimulatorApp\SimulatorApp.csproj -c Release -r win-x64 --self-contained true -o E:\SimulatorApp\publish-full
```
